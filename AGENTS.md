### 📋 高階離散統計引擎：全域編程鐵律 (Global Architectural Spec)

#### 1. 🛑 【Don't】盲目信任全域與閉包變數的隱式生命週期
*   **硬傷回顧**：命中軌道與傷害軌道在各自執行解構時，同時使用了高度通用的變數名。在同一檔案連續執行多組測試或存在閉包共享時，前置函數的物理位移與快取機率，直接無預警地覆蓋到了傷害 Pipeline 的收尾變數中，造成數值污染。
*   **全域鐵律 (Do)**：在任何涉及複數流水線（Pipelines）或動態解構（Destructuring）的環境中，必須執行嚴格的語義命名隔離，絕不使用通用名。必須顯式宣告為獨立前綴變數，在編譯/直譯層面建立作用域防火牆。

#### 2. 🛑 【Don't】將「相對座標」與「絕對座標」進行盲目混合代數加總
*   **硬傷回顧**：這是導致極端環境下傷害數據產生幽靈偏移的真兇。由於摺積算法具備高效的「切除前端零點」優化，導致返回的機率矩陣其首項索引已被向左推成了相對座標（代表該骰子的實質最小值）。此時如果直接在迴圈內將索引當成實質點數相加，本質上是把相對索引直接拿來做絕對加法，導致數值在時域整體錯位。
*   **全域鐵律 (Do)**：當資料結構包含自動裁剪與平移優化時，任何時域遍歷加總，必須強制引入實質原點位移補償（Offset Compensation）。陣列索引只負責提供「相對步進」，實質的物理坐標必須由追蹤的整數位移進行絕對校正。

#### 3. 🛑 【Don't】在時域直接執行「非連續性、非線性的硬編碼剪枝」
*   **硬傷回顧**：為了在進場前隔離特定特例（如大失敗），若盲目地在時域將該點機率直接清零，會導致機率陣列的總機率和縮水跌落 1.0，進而觸發了後置算法結尾的防漂移歸一化校正，硬生生把所有正常點的機率等比例放大，人為扭曲了分佈波峰。
*   **全域鐵律 (Do)**：機率質量守恆（總機率和恆等於 1.0）是離散統計的底線。所有基於機制的特例保底與剪枝，絕對不允許提前破壞輸入源的分佈結構，必須在最終的判定遍歷（Decision Traversal）中進行末端剔除。

#### 4. 🛑 【Don't】容忍浮點數微觀殘留與「巨觀布林邏輯」進行等號對撞
*   **硬傷回顧**：理論上某一資源在消耗完畢後的殘留計算中，減法算出的消耗機率應該精確等於 0，進而關閉特定分支。但因為前置變換的數萬次連環複數乘除，機率在硬體暫存器中產生了極微小的二進位制裂縫（如 0.999999999999999），導致減法算出了極小正數，強行啟動了「陰屍路分支」，把不該存在的常規傷漏進了分佈中。
*   **全域鐵律 (Do)**：在多層次高階摺積環境下，硬體捨入誤差會隨摺積層數呈指數級放大。所有涉及大於 0 或不等於 0 的巨觀邏輯門檻控制，必須統一改用全域巨觀門檻常數進行硬性剪枝過濾，用巨觀防線直接在時域踩死微觀的二進位制底噪。

#### 5. ✔️ 【Do】為「空操作 / 單位元」建立時域直通車機制 (Pass-through)
*   **核心價值**：當系統沒有額外資源、或 Pool 中只有單一分佈時，如果不做限制，它依然會帶著長度為 1、索引 0 機率為 1.0 的單位脈衝信號強行進入頻域相乘。這不僅白白浪費了蝴蝶迭代開銷，更會在逆向還原歸一化時，把旋轉因子產生的二進位制硬傷底噪放大數萬倍，浮出水面。
*   **全域鐵律 (Do)**：在頻域摺積架構中，對於單一元素、空物件或數學上的常數單位元（Identity Element），必須在進場前執行「時域直通車優化」攔截，直接深拷貝克隆該分佈並直通返回。開銷直接歸零，且 100% 封殺旋轉因子引發的頻域噪點回繞。

#### 6. ✔️ 【Do】實施「統計分佈」與「代數解析」的雙軌驗證架構
*   **核心價值**：在處理極端低機率區間（例如極端連環爆擊且骰面全滿，機率低於 10 的負 20 次方）時，任何高階離散統計工具在有限位數下都必然會面臨門檻剪枝的衝擊，導致從統計矩陣截取出來的長度與理論上限不符。
*   **全域鐵律 (Do)**：不要試圖從已經歷多次門檻剪枝和抹平的統計質量分佈（PMF/CDF）中直接逆向獲取極值數據。極限邊界（例如全輪理論最大傷害）必須在流水線開頭，利用純代數（Algebraic）解析迴圈獨立加總，作為細節欄位傳回，以確保回傳的理論上限不論在多麼極端的環境下，都能展現出 100% 穩定且不偏不倚的絕對真理邊界。

### 📋 高階離散統計引擎：全域自動化測試與遙測鐵律 (Global Architectural Spec - Test & Telemetry Addendum)

#### 7. 🛑 【Don't】在單元測試中使用二進位等號對撞與盲目座標跨度硬編碼
*   **硬傷回顧**：在多段打擊的跨輪管線測試中，若誤認為「除去落空機率後，所有傷軌質量必定大於雙保底起點（102點）」，會直接抹殺了二項分佈展開中的交叉項（即恰好 1 擊命中、落在 51~74 點的 9.5% 質量），導致測試把合法的傷軌點位誤判為「位移流失」，強行噴出中斷。同時，若在機率檢驗中直接與浮點數對撞，會因暫存器微觀裂縫導致測試集在特定硬體上產生幽靈失敗。
*   **全域鐵律 (Do)**：所有機率與數據比對必須依循**三軌隔離防線**。第一軌（命中率）必須統一改用微觀硬體精度常數進行容差安全閥判定（`Math.abs(actual - expected) < MACHINE_EPSILON`）。第二軌（時域真空區）必須根據多項式展開的實質狀態樹，嚴格推導出「絕對真空區間」，並檢驗該區間的污染質量是否恆小於巨觀時域抹平門檻（`<= BUCKET_EPSILON`）。第三軌（代數上限）必須與管線開頭的純代數解析軌道進行強行對齊閉合。

#### 8. ✔️ 【Do】落實「未命中質量不連續性」與時域二次物理隔離機制
*   **核心價值**：單擊未命中（`pMiss`）在 DND 5e 中代表造成 0 點傷害，它本質上是一個不連續的狄拉克脈衝信號（Delta Impulse）。若為了維持傅立葉前端裁剪優化而中途將單擊 PDF 除以 `pAnyHit` 進行盲目歸一化，會導致多段打擊在頻域捲繞相乘時，完全流失 1 命中、2 命中的二項展開狀態軌道，使中間傷軌在時域中憑空蒸發。
*   **全域鐵律 (Do)**：單擊 PDF 內部的第 0 格**必須允許 pMiss 質量的原位存活**（用以承接遊戲實質未命中與大失敗強制落空），但同時管線內部**必須顯式接收並平移疊加單輪與跨輪摺積返回的實質位移補償量（roundOffset 與 finalTotalOffset）**。藉由將 `pMiss > 0` 的零點桶位與保底原點追蹤完美倂行，讓多層次摺積在時域中自然展開出完整的二項狀態樹，徹底阻斷機率塌陷與向左幽靈平移。

#### 9. ✔️ 【Do】執行「遙測日誌大一統規範（Unified Logging Pattern）」以實現無噪點自動化回歸斷言
*   **核心價值**：當系統歷經多輪重構或算法優化時，缺乏統一日誌格式與對齊上下文的測試集會增加開發者的認知開銷，且無法一鍵嵌入 CI/CD 腳本中執行程式碼級的無噪點斷言檢查。
*   **全域鐵律 (Do)**：所有自動化測試群組必須採用標準的樹狀視覺拓撲與語義標籤對齊。群組啟動時統一使用 `🧪 [啟動測試群組] -> 函式名` 為起手式；內部檢查點通過時統一採用 `  ├─ ✅ Case/檢查點 編號 通過: [核心機制描述] ── 數據對齊完全無偏誤。` 的分支標記；斷言失敗時必須以 `[失敗] 錯誤原因描述 (預期 X, 實測 Y)` 格式噴出完整上下文；群組完全通行時，底部必須輸出由雙橫線包裹的通行證（`🎉 [群組通關] ── 函式名: [防線描述] 100% 守恆！`）。不論管線內部經歷多麼複雜的頻域與時域變換，其外部測試防線必須呈現出工業級的確定性。
